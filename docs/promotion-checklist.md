# Checklist de promotion Lab → Core

Une brique/idée ne passe dans Afkar Core que si TOUT est coché :

- [ ] Documentation propre et à jour (pas un stub, un vrai README d'usage)
- [ ] ADR qui justifie la décision (contexte, alternatives, conséquences)
- [ ] Testé en usage réel (pas juste en démo) — préciser depuis quand/combien de fois
- [ ] Revu par l'utilisateur (pas juste construit et oublié)
- [ ] Passe la [tool-adoption-checklist](tool-adoption-checklist.md) si ça
      introduit une dépendance externe

Si un point manque : ça reste dans le Lab, pas de promotion prématurée.
